# 見守り機能 UI/UX ハンドオフ資料（別AI/デザイナー向け）

> 作成日: 2026-03-06
> 目的: UI/UXを別AIに委任するための、画面要件・制約・文言・成果物定義
> 実装仕様の参照: `docs/wellness-check-delivery-spec.md`

---

## 0. この資料の使い方

- この資料は「見守り機能のUI/UXを設計する担当」にそのまま渡すことを想定している。
- 実装担当は本資料の画面仕様を受けて、コンポーネント実装に落とす。
- 最終ビジュアルは既存トーン（`app/client/src/app.css` のカラートークン）を維持する。

---

## 1. UXコンセプト

### 1.1 絶対に守る方向性

- 「監視」ではなく「日々のおしゃべり支援」として見せる。
- 本人が主導権を持つUIにする。
- 家族向けは“安心の把握”が主で、通知過多にしない。

### 1.2 避けるべき表現

- 「監視」「追跡」「異常検知中」などの強い語
- 「見守りをONにしないと危険」など不安訴求
- 家族主導で本人設定を変更できる印象

---

## 2. 対象ユーザー

### 2.1 本人（高齢者）

- 読みやすさ最優先（既存 font-size 3段階を尊重）
- 選択肢は少なく、文章は短く
- 設定後に「今なにが共有されるか」が即わかること

### 2.2 家族

- 短時間で状況が把握できること
- 毎日通知ではなく、週次サマリー + 必要時通知

---

## 3. 必須画面一覧

### S1. 見守り機能 有効化ダイアログ（本人）

目的:
- 目的説明、同意取得、共有プリセット初期選択

必須要素:
- タイトル
- 1段落説明（監視ではないことを明示）
- 共有プリセット3択
- 同意チェック
- `始める` / `あとで` ボタン

状態:
- 初期
- 入力不足（同意未チェック）
- 送信中
- エラー

### S2. 見守り設定セクション（Settings画面内）

目的:
- 有効/停止、共有範囲、週次配信日、エスカレーション閾値を編集

必須要素:
- トグル: 見守りを使う
- 共有プリセット選択
- 配信曜日選択
- 一時停止ボタン
- 現在共有中の情報サマリー
- 最終更新日時

状態:
- enabled
- disabled
- paused
- save success
- save error

### S3. 本人向けステータスカード（会話/設定のいずれか）

目的:
- 本人に「最近の状況」と「共有中の範囲」を可視化

必須要素:
- 直近7日会話回数
- 連続未接触日数
- 共有レベル表示
- `共有内容を変更` 導線

### S4. 家族向け週次サマリーカード（Family画面）

目的:
- 家族が1スクリーンで今週の状態を把握

必須要素:
- 今週の接触日数
- 現在ステータス（安定/注意）
- 1〜2行要約
- 変化点タグ（任意）
- 前週比（任意）

状態:
- loading
- no data
- stable
- warning
- urgent

### S5. 異常通知詳細ビュー（Notification詳細）

目的:
- 通知を見た家族が次に取る行動を迷わない

必須要素:
- 何が起きたか（連続未接触○日）
- 最終接触日時
- 推奨アクション（電話してみる等）
- `家族メモ`（任意）

---

## 4. 画面間フロー

### F1. 本人オンボーディング

1. Settingsで見守りカード表示
2. `見守りを始める`
3. 有効化ダイアログ（S1）
4. 同意 + プリセット選択
5. 完了トースト + S2へ

### F2. 本人が共有範囲を変更

1. S2で共有レベル変更
2. 確認メッセージ
3. 保存成功
4. S3に反映

### F3. 家族が週次サマリーを確認

1. Family画面に入る
2. S4表示
3. 詳細導線タップ
4. 必要時のみ通知詳細（S5）へ

---

## 5. マイクロコピー案（日本語）

### 5.1 本人向け

- 見守り説明: `毎日のおしゃべり状況を、あなたが決めた範囲でご家族に伝えます。`
- 非監視説明: `監視する機能ではありません。共有内容はいつでも変更・停止できます。`
- 共有レベル:
- `基本だけ` / `会話回数などの基本情報のみ共有します`
- `要約も共有` / `基本情報に加えて、短い週間まとめを共有します`
- `詳細共有` / `必要な変化点も含めて共有します`
- 一時停止: `しばらく見守りをお休みする`

### 5.2 家族向け

- 安定時: `今週は安定して会話できています。`
- 注意時: `2日連続で会話がありません。ご連絡をおすすめします。`
- 強注意時: `3日以上会話が確認できていません。早めのご確認をお願いします。`

## 6. デザイン制約（実装都合）

### 6.1 技術・画面構成

- 既存ナビゲーションは `conversation` `note` `history` `settings` `family-dashboard` を使用
- 既存コンポーネント構成を活かす（大規模なルーター変更は避ける）
- 主要組み込み先:
- `SettingsScreen.tsx`
- `FamilyScreen.tsx`
- `NotificationList.tsx`

### 6.2 トークン/タイポ

- カラーは既存CSSトークンを優先利用
- `--color-bg-primary` `--color-bg-surface` `--color-accent-primary`
- `--color-text-primary` `--color-text-secondary`
- フォントは `Noto Sans JP` を維持
- 既存 `data-font-size` 3段階への対応必須

### 6.3 アクセシビリティ

- タップ領域最小44px
- 文字だけで状態を伝えない（色 + 文言）
- 重要変更は確認ステップを入れる
- 音声操作利用を想定し、ラベルは短く明快に

## 9. デザイン成果物の定義（別AIに依頼する内容）

以下を納品対象として依頼する。

1. 主要5画面のワイヤーフレーム（モバイル/デスクトップ）
2. 主要状態（loading/empty/error/success/paused）の差分UI
3. コンポーネント一覧（カード、プリセット選択、通知セル）
4. 画面遷移図（F1-F3）
5. マイクロコピー確定版
6. エンジニア向け実装注記（余白、文字サイズ、状態切替ルール）

## 10. 別AIに渡す依頼テンプレート（コピペ用）

```text
あなたはこのプロダクトのUI/UXデザイナーです。
以下の条件で、見守り機能のUI/UXを設計してください。

[目的]
- 監視感を出さずに、本人主権の見守り体験を作る
- 家族は週次で安心を把握し、異常時だけ即時に気づける

[対象画面]
- 見守り有効化ダイアログ
- Settings内の見守り設定セクション
- 本人向け見守りステータスカード
- 家族向け週次サマリーカード
- 異常通知の詳細ビュー

[必須制約]
- フォント: Noto Sans JP
- 高齢者向けで文字大きめ・行間広め
- 3段階の共有プリセットをわかりやすく表現
- loading/empty/error/success/paused 状態を全画面で定義
- 既存情報設計（settings/family-dashboard）を大きく壊さない

[納品形式]
- 画面別ワイヤー
- 状態差分
- 画面遷移図
- 実装注記（コンポーネント単位）
- マイクロコピー最終案
```

## 11. デザインレビュー観点（受け入れ基準）

- 本人が「何を共有しているか」を3秒以内で理解できるか
- 一時停止までの操作が3タップ以内か
- 家族がサマリーを10秒以内で読めるか
- 注意/強注意の差が色以外でも伝わるか
- 監視的な語彙・印象になっていないか
